Akka is a toolkit and runtime for building highly concurrent, distributed, and fault tolerant event-driven applications on the JVM. Akka can be used with both Java and Scala. One of the most powerful features of Akka is it's Actor model of concurrency.

Explore the Code

The app directory contains the application's Java and Scala source code. A few example Actor-based applications are provided. The HelloAkkaJava.java and HelloAkkaScala.scala files send a message to an Actor every 5 seconds. This message is then printed to the console.
The PiJava.java and PiScala.scala files provide Actor-based examples for computing Pi.

Run the App

In Run, select the application to run from the down-down list, and select Start. The Pi examples will do their computation and then exit. The Hello examples will continue running until stopped.

Test the App

The Pi Java and Scala examples each have unit tests in the test directory. The Java tests use JUnit, while the Scala tests use ScalaTest. Making changes to the application's source or the test source will cause the tests to be automatically re-run.
